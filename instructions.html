How to Create an "About Me" Page on GitHub Pages

Table of Contents

Step 1: Create a GitHub Account & Repository

Welcome! We're about to dive into creating your very own personal website using GitHub Pages. Don’t worry if this feels overwhelming — we're here to guide you step by step. Let’s start by creating a GitHub account (if you don’t have one) and your first repository.

Creating an Account: If you don’t already have a GitHub account, go to GitHub.com and sign up. It's free! Once you've registered, log in to your new account.

Creating Your Repository: Now, click on the “New Repository” button (you can find it on your dashboard after logging in).

Important: You must name your repository like this: yourusername.github.io, replacing "yourusername" with your actual GitHub username. This exact name is how GitHub knows you’re creating a website.

Tip: Make sure the repository is set to public, so your website will be visible to everyone!

Step 2: Set Up GitHub Pages

Great job creating your repository! Now let’s set it up as a GitHub Pages site.

Go to your repository’s settings by clicking on the “Settings” tab. Scroll down until you find the "GitHub Pages" section.

Under “Source”, select the main branch (this is where all your code and changes will go).

Be patient: It might take up to 5 minutes for your new site to become available online. Refresh the page after a few minutes to check.

Step 3: Customize Your 'About Me' Page

Now for the fun part — let's add some content to your website! You’ll need to edit the default index.html file. Think of this as the homepage for your site, where you’ll introduce yourself.

To start, click on the index.html file in your repository. Now, replace the placeholder text with your own information.

Pro Tip: Make sure to replace the "username" placeholders with your actual GitHub username.

Here’s an example code that you can use to create a simple 'About Me' page. Just copy and paste the following HTML code into your index.html file:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>About Me - [Your Name]</title>
</head>
<body>
    <header>
        <h1>About Me</h1>
    </header>

    <section>
        <h2>Hello! I'm [Your Name]</h2>
        <p>
            Welcome to my personal page! I am a [your profession or hobby] who loves 
            [something you're passionate about]. This website is a small introduction 
            to who I am and what I do.
        </p>
    </section>

    <section>
        <h2>Connect with me</h2>
        <ul>
            <li><a href="https://github.com/yourusername" target="_blank">GitHub</a></li>
            <li><a href="https://linkedin.com/in/yourusername" target="_blank">LinkedIn</a></li>
            <li><a href="https://twitter.com/yourusername" target="_blank">Twitter</a></li>
        </ul>
    </section>

</body>
</html>

This is just a starting point! Feel free to customize it as you wish by adding more sections, updating the links, or even styling it with CSS later.

Remember to organize your content with headings (like <h1> and <h2>) and bullet points to keep it neat and readable.

Step 4: Commit and Publish

Ready to see your changes live? You’re almost there! After editing your index.html file, scroll to the top and click the “Commit changes” button.

Make sure to add a meaningful commit message (e.g., “Updated About Me page”) to keep track of changes.

Once committed, it may take a minute or two for the changes to appear on your live site. Just refresh your browser after a bit to see the magic happen!

Troubleshooting

If something’s not working quite right, don’t panic! Here are a few common issues and how to solve them:

  • Website not showing up: Wait a few minutes, then refresh. It can take time for GitHub Pages to publish the site.
  • Changes not appearing: Double-check that you’ve committed your changes to the correct branch (usually main).
